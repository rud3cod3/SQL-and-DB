### NoSQL
- Designed to handle large Volume of Databases both structured and unstructured
- High user loads
- Need for flexibility and flexibility

### Types of NoSQLDB
* Stored in documents ( usually json, bson and xml )
 - Each document is self-contained and can have a unique structure
 - Ideal for hierarchical data storage

* Key-value stores
 - Store data as a collection of key-pairs
 - The key is uniq, and the value can be anything
 - Simple and fast, like a dictionary

* Columna (wide column) Databases
 - Store data in columns instead rows
 - Great for large datasets with lots of attributes
 - Often used for analytics and time-series data

* Graph database
 - Designed for storing relationships between entities
 - USe nodes, edges and properties
